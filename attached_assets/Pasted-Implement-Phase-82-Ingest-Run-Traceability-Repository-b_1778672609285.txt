  Goal:
  Make /studio upload ingest runs traceable across:
  1. Hagen analyze
  2. Hagen enrich
  3. CM save to library
  4. Optional assignment to customer
  5. Fire-and-forget humor enrich

  Required work:
  1. In api-server ingest run typing:
     - Update IngestRunStage in artifacts/api-server/src/lib/ingest-runs.ts if needed to include assignment tracking,
  preferably `assigned`.
     - Do not invent a large new state machine. Keep existing statuses.

  2. In artifacts/api-server/src/routes/studio.ts:
     - Import HAGEN_CONTRACT_VERSION from upstream-proxy.
     - When analyze starts, write hagen_contract_version to the ingest run.
     - When analyze succeeds, write:
       - hagen_request_id scalar from fetchHagenJson result.requestId
       - hagen_contract_version scalar
       - mergeResult.analyze_summary.request_id
       - mergeResult.analyze_summary.hagen_contract_version
       - existing gcs_uri and has_analysis

  change the top-level completed status after save.

       - On successful insert, update ingest_runs non-fatally:
